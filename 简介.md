## 1. 项目概述

### 1.1 项目定位

`pr-player` 是一个**纯浏览器端**视频流媒体解析与播放库，核心目标是**不依赖 MSE（Media Source Extensions）**，直接基于 WebCodecs API 实现视频流的解复用、解码与渲染。

### 1.2 核心能力

| 能力 | 说明 |
|------|------|
| **多协议支持** | FLV / HLS / DASH / MP4 四种流媒体格式 |
| **WebCodecs 解码** | 基于浏览器原生 `VideoDecoder` / `AudioDecoder` |
| **MediaStream 输出** | 直接生成 `MediaStream`，可接入 `<video>` 标签 |
| **多路裁剪** | 基于 SEI 数据的实时视频区域裁剪，每路独立输出 |
| **多线程架构** | Demuxer / Decoder / Render 各自运行在独立 Worker |
| **直播优化** | 内置追帧（Frame Track）机制，自动检测缓冲区积压 |
| **全链路钩子** | 解复用/解码/渲染全链路可插桩回调 |

### 1.3 技术栈

```
┌──────────────────────────────────────────────┐
│  浏览器 API                                    │
│  ├── WebCodecs (VideoDecoder/AudioDecoder)    │
│  ├── Media Stream API (TrackGenerator)        │
│  ├── Web Audio API (AudioContext)             │
│  ├── Web Workers (多线程)                      │
│  └── Streams API (ReadableStream)              │
├──────────────────────────────────────────────┤
│  运行时依赖                                     │
│  ├── pr-audio-stream (音频流封装)               │
│  └── pr-fetch (HTTP 拉流)                      │
├──────────────────────────────────────────────┤
│  开发工具                                       │
│  ├── TypeScript 6.0                            │
│  ├── Vite 8.0 (库模式构建)                      │
│  └── vite-plugin-dts (类型声明生成)             │
└──────────────────────────────────────────────┘
```

## 2. 架构设计

### 2.1 整体架构图

```
                        ┌──────────────────┐
                        │   PrPlayer (主控)  │
                        │   - 协议选择       │
                        │   - 管线编排       │
                        │   - 事件回调       │
                        └────────┬─────────┘
                                 │
              ┌──────────────────┼──────────────────┐
              ▼                  ▼                  ▼
    ┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
    │ DemuxerWorker   │ │ DecoderWorker   │ │ RenderWorker    │
    │ (Web Worker)    │ │ (Web Worker)    │ │ (Web Worker)    │
    │                 │ │                 │ │                 │
    │ FLV → ParseFLV  │ │ VideoDecoder    │ │ 主路输出         │
    │ HLS → ParseTS   │ │ AudioDecoder    │ │ 裁剪路输出 ×N    │
    │ MP4 → ParseFMP4 │ │ SEI 解析        │ │                 │
    │ DASH→ ParseFMP4 │ │ 追帧控制        │ │                 │
    └─────────────────┘ └─────────────────┘ └─────────────────┘
              │                  │                  │
              ▼                  ▼                  ▼
         EncodedChunk     VideoFrame           MediaStream
                          AudioData          ─────────────►
                              │               <video> 标签
                              ▼
                    ┌─────────────────┐
                    │  AudioPlayer    │
                    │  (主线程)        │
                    │                 │
                    │  AudioData →    │
                    │  AudioBuffer →  │
                    │  Web Audio API  │
                    └─────────────────┘
```

### 2.2 设计原则

1. **分层解耦**: 解复用 → 解码 → 渲染 → 音频播放，各层通过消息传递通信
2. **Worker 隔离**: 计算密集型操作全部卸载到 Worker 线程，保证主线程流畅
3. **Transferable 优化**: VideoFrame / AudioData 通过零拷贝传输
4. **协议适配器模式**: 不同容器格式对应独立 Parser，统一输出接口
5. **渐进增强**: `MediaStreamTrackGenerator` 不可用时自动降级为 `canvas.captureStream()`

### 3.2 PrPlayer — 主播放器

**核心职责**: 拉流控制、管线编排、事件回调分发。

**关键属性**:

| 属性 | 类型 | 说明 |
|------|------|------|
| `flv` | 私有方法 | FLV 拉流循环（HTTP 长连接，循环读取 Reader） |
| `hls` | 私有方法 | HLS 拉流循环（解析 .m3u8 → 逐个拉取 TS 分片） |
| `mp4` | 私有方法 | MP4 渐进式下载（一次性 Reader） |
| `dash` | 私有方法 | DASH 自适应流（解析 .mpd → SegmentTemplate / SegmentList + ByteRange） |
| `cut` | 命名空间 | 多路视频裁剪输出管理 |
| `on` | 方法 | 注册事件回调（demuxer / decoder / error） |

**初始化流程**:

```
init()
  ├── 1. 初始化 Render (渲染器): initSuccessMap 按 renderId 等待就绪
  ├── 2. 初始化 Decoder (解码器): 接收 VideoFrame/AudioData/SEI
  ├── 3. 初始化 Demuxer (解复用器): 根据 pattern 选择 Parser
  └── 4. 初始化 AudioPlayer (音频播放器): 关联 AudioContext
```

**追帧机制** (`setFrameTrack`):

```
积压帧数 > frameTrack[1]  →  加速解码 (decodingSpeedRatio > 1)
积压帧数 < frameTrack[0]  →  恢复正常解码速率
```

## 4. 数据流与协议解析

### 4.1 数据管线

```
网络流 (ReadableStream)
    │
    ▼
Cacher (缓冲管理)
    │ 累积数据块，处理跨块边界
    ▼
Demuxer (协议解复用)
    │
    ├── 回调: onDemuxerInfo(data)     → 流元信息 (宽高/编码/SEI等)
    ├── 回调: onDemuxerConfig(data)   → 音视频解码配置
    └── 回调: onDemuxerChunk(data)    → EncodedVideoChunk / EncodedAudioChunk
            │
            ▼
        Decoder (WebCodecs)
            │
            ├── 回调: onDecoderVideo(data)  → VideoFrame
            ├── 回调: onDecoderAudio(data)  → AudioData
            ├── 回调: onDecoderSei(data)    → SEI 解析数据
            └── 回调: onDecoderAnalysis(data) → 解码分析 (FPS/速率等)
                    │
                    ▼
                Render (渲染输出)
                    │
                    ├── 主路: MediaStreamTrackGenerator.writable → MediaStream
                    └── 裁剪路: OffscreenCanvas → canvas.captureStream()
```

### 4.2 协议解析器对照

| 格式 | 解析器 | 容器 | 视频编码封装 | 音频编码封装 | 关键解析步骤 |
|------|--------|------|-------------|-------------|-------------|
| **FLV** | `flv264Parser.ts` | FLV Tag | H.264 AVCC | AAC raw | Header → Script Tag(AMF) → Audio Tag → Video Tag |
| **HLS** | `ts264Parser.ts` | MPEG-TS | H.264 AnnexB | AAC ADTS | PAT → PMT → PES → NALU / ADTS frame |
| **MP4** | `fmp4Parser.ts` | ISO BMFF | H.264 AVCC | AAC | moov(trak/stbl/stsd) → moof(trun) → mdat |
| **DASH** | `mpdParser.ts` + `fmp4Parser.ts` | fMP4 分片 | H.264 AVCC | AAC | MPD(XML) → AdaptationSet → SegmentTemplate → 逐片解析 |

### 4.3 关键解析细节

**FLV 协议**:

```
FLV Header (9 bytes)
  ├── Signature: "FLV" (3 bytes)
  ├── Version: 1 (1 byte)
  ├── TypeFlags: audio/video (1 byte)
  └── DataOffset: 9 (4 bytes)

PreviousTagSize (4 bytes) ─ 前一个 Tag 的大小

FLV Tag
  ├── TagType: 8=audio, 9=video, 18=script (1 byte)
  ├── DataSize (3 bytes)
  ├── Timestamp + TimestampExtended (4 bytes)
  ├── StreamID (3 bytes)
  └── Data
      ├── Script Tag → AMF 解析 → onMetaData (宽高/帧率/编码)
      ├── Video Tag → AVCDecoderConfigurationRecord → NALU
      └── Audio Tag → AudioSpecificConfig → AAC raw frame
```

**MPEG-TS 协议**:

```
TS Packet (188 bytes)
  ├── Header (4 bytes): sync_byte(0x47) + PID + ...
  └── Payload (184 bytes)

PAT (PID=0x0000) → program_number → PMT_PID

PMT → stream_type → elementary_PID
  ├── stream_type=0x1B → H.264 video
  └── stream_type=0x0F → AAC audio

PES Packet (重组自多个 TS Packet)
  ├── PES Header
  └── PES Payload
      ├── Video: AnnexB NAL Units (00 00 00 01 start code)
      └── Audio: ADTS Header + AAC raw frame
```

**ISO BMFF (MP4/fMP4)**:

```
ftyp (file type)
moov (movie metadata)
  ├── mvhd (movie header)
  └── trak (track)
      ├── tkhd (track header)
      └── mdia (media)
          ├── mdhd (media header)
          ├── hdlr (handler - vide/audio)
          └── minf (media information)
              └── stbl (sample table)
                  ├── stsd (sample description - AVCC/ESDS)
                  ├── stts (time-to-sample)
                  ├── ctts (composition time offset)
                  ├── stss (sync sample)
                  ├── stsz (sample size)
                  └── stco/stco64 (chunk offset)

moof (movie fragment - fMP4)
  ├── mfhd (fragment header)
  └── traf (track fragment)
      ├── tfhd (track fragment header)
      ├── tfdt (decode time)
      └── trun (sample info: size/duration/flags)

mdat (media data) ─ 实际的编码数据
```

## 5. 多线程架构

### 5.1 Worker 分工

| Worker | 文件 | 实例化类 | 职责 |
|--------|------|---------|------|
| DemuxerWorker | `demuxer.worker.ts` | `Demuxer` | 协议解析、数据解复用 |
| DecoderWorker | `decoder.worker.ts` | `Decoder` | 音视频解码、SEI 提取 |
| RenderWorker | `render.worker.ts` | `Render` | 视频帧渲染、多路输出 |

### 5.2 通信机制

```
主线程                         Worker 线程
  │                                │
  │  postMessage({ type, data })   │
  │ ─────────────────────────────► │
  │                                │
  │  postMessage({ type, data })   │
  │ ◄───────────────────────────── │
  │                                │
  │  Transferable 零拷贝:           │
  │  VideoFrame, AudioData          │
  │ ◄───────────────────────────── │
```

各 Worker 通过 `?worker&inline` 内联到构建产物中，无需额外文件加载。

### 5.3 Worker 封装层

每个 Worker 都有对应的封装类 (`DemuxerWorker.ts`, `DecoderWorker.ts`, `RenderWorker.ts`)：

- **Promise 化**: 将 `postMessage` / `onmessage` 包装为 Promise，支持 `await` 调用
- **类型安全**: TypeScript 泛型约束消息类型
- **生命周期管理**: `init()` / `close()` 控制 Worker 创建与销毁
- **Transferable 自动识别**: 自动识别 VideoFrame/AudioData 等 Transferable 对象

## 6. 渲染与输出

### 6.1 渲染模式

| 模式 | API | 适用场景 | 性能 |
|------|-----|---------|------|
| **Generator 模式** (首选) | `MediaStreamTrackGenerator` + `writable.getWriter()` | 浏览器支持 WebCodecs | 零拷贝，最佳性能 |
| **Canvas 模式** (降级) | `OffscreenCanvas` + `canvas.captureStream()` | Generator 不可用时 | 有一次 GPU→CPU→GPU 拷贝 |

### 6.2 多路裁剪输出

通过 `cut` 命名空间支持创建多路独立的裁剪输出：

```typescript
// 示例
const cutStream = player.cut.create({
  renderId: 'cut-1',
  visibleRect: { x: 0, y: 0, width: 640, height: 360 }
})

// <video srcObject={cutStream} />
```

裁剪原理：在 Render 中为每个 `renderId` 维护独立的 `VideoFrame` 处理管线，通过 `VideoFrame` 构造函数的 `visibleRect` 参数实现区域裁剪。

### 6.3 音频播放

音频不经过 Render Worker，在主线程通过 Web Audio API 处理：

```
AudioData (WebCodecs)
    │
    ▼
AudioBuffer (Web Audio)
    │  支持格式: s16, s16-planar, f32, f32-planar, u8, u8-planar
    │
    ▼
AudioBufferSourceNode
    │  playbackRate / detune 控制变速
    │  nextStartTime 精确调度避免重叠
    │
    ▼
MediaStreamAudioDestinationNode
    │
    ▼
pr-audio-stream (合并到最终 MediaStream)
```

## 7. API 设计

### 7.1 初始化

```typescript
const player = new PrPlayer({
  url: 'https://example.com/stream.flv',    // 流地址
  autoPlay: true,                            // 是否自动播放
  renderId: 'main',                          // 默认渲染通道ID
  frameTrack: [30, 50],                      // 追帧阈值 [最小, 最大]
  useCanvasRender: false,                    // 是否强制使用 Canvas 渲染
})
```

### 7.2 事件回调

```typescript
// 解复用层回调
player.on('demuxer', (type, data) => {
  switch (type) {
    case 'info':   // 流元信息: 宽高/编码/SEI业务数据
    case 'config': // 音视频解码配置
    case 'chunk':  // 编码数据块
  }
})

// 解码层回调
player.on('decoder', (type, data) => {
  switch (type) {
    case 'video':    // VideoFrame
    case 'audio':    // AudioData
    case 'sei':      // SEI 解析结果
    case 'analysis': // 解码分析: FPS/速率/积压
  }
})

// 错误回调
player.on('error', (error) => {
  console.error(error)
})
```

### 7.3 裁剪命名空间

```typescript
// 创建裁剪路
player.cut.create({
  renderId: 'cut-1',
  visibleRect: { x: 100, y: 50, width: 320, height: 240 },
  autoPlay: true,
})

// 修改裁剪区域
player.cut.modify('cut-1', {
  visibleRect: { x: 200, y: 100, width: 640, height: 480 },
})

// 暂停/恢复裁剪路
player.cut.pause('cut-1')
player.cut.resume('cut-1')

// 移除裁剪路
player.cut.remove('cut-1')

// 获取裁剪路 MediaStream
const stream = player.cut.getStream('cut-1')
```

### 7.4 控制方法

```typescript
player.start()                          // 开始拉流
player.pause()                          // 暂停
player.resume()                         // 恢复
player.destroy()                        // 销毁所有资源
player.setFrameTrack([min, max])        // 设置追帧阈值
player.getMediaStream()                 // 获取主路 MediaStream
```

## 10. 技术决策与权衡

### 10.1 为什么不使用 MSE？

| MSE 方案 | WebCodecs + TrackGenerator 方案 |
|----------|-------------------------------|
| 依赖浏览器封装好的 Codec 实现 | 直接调用底层 Codec |
| 无法获取解码后的原始帧 | 可获得 VideoFrame/AudioData |
| 无法实现帧级裁剪 | 支持 visibleRect 裁剪 |
| 无法提取 SEI 数据 | 可从 NALU 中解析 SEI |
| MSE 的 Buffer 管理复杂 | 完全自主控制管线 |

### 10.2 为什么用 Worker 多线程？

- **解复用**: FLV/TS/fMP4 协议解析是纯 CPU 计算，不阻塞主线程
- **解码**: WebCodecs 在 Worker 中运行可获得更好的调度优先级
- **渲染**: OffscreenCanvas 只能在 Worker 中使用
- **Transferable**: VideoFrame/AudioData 零拷贝传输，无性能损失

### 10.3 为什么不支持 RTMP？

RTMP 是 TCP 长连接协议，浏览器不支持原生 TCP Socket。替代方案：
- 服务端转协议：RTMP → HTTP-FLV / HLS
- WebSocket 转发：RTMP → WebSocket-FLV

### 10.4 当前局限

1. **仅支持 H.264 + AAC**: 未实现 H.265/AV1/Opus 等编码格式
2. **无加密流支持**: 不支持 DRM/AES 解密
3. **无明显音画同步策略**: 依赖 Web Audio API 的精确调度，未实现主动同步校正
4. **无自适应码率**: DASH 支持解析 MPD，但未实现多码率切换逻辑
5. **浏览器兼容性**: 依赖 WebCodecs API（Chrome 94+, Edge 94+），Firefox/Safari 不支持

### 10.5 后续演进方向

- 支持 H.265 (HEVC) 编码
- 加密流解密 (AES-128 for HLS)
- 自适应码率 (ABR) 逻辑
